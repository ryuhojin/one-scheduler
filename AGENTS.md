# AGENTS.md - one-line 프로젝트 실행 가이드

## 1) 프로젝트 목표
`one-line`은 사내 일정 등록/공유를 위한 서비스다.
제품은 2개 클라이언트로 구성한다.
- 사용자용: Electron PC 앱
- 관리자용: Admin Web 페이지

핵심 목표는 다음 3가지다.
- 권한 범위에 맞는 일정 공유(회사/부서/그룹/개인)
- 신규 일정 등록 시 실시간 반영
- 공유 대상 사용자에게 PC 알림 제공

## 2) 도메인 규칙(확정)
일정 분류 및 공개 범위는 아래 규칙을 고정한다.
- 회사일정: 같은 회사 소속 전체에 공유
- 부서일정: 같은 부서 소속 사용자에게 공유
- 그룹일정: 그룹 멤버에게 공유(그룹 생성/초대/탈퇴/강퇴 지원)
- 개인일정: 기본 비공개, 명시적으로 선택한 사용자에게만 공유 가능

추가 공통 규칙
- 공유받은 일정이라도 본인이 수정 권한이 없으면 읽기 전용
- 일정 생성자는 기본적으로 수정/삭제 권한 보유
- 모든 일정은 생성/수정/삭제 이력을 남긴다(감사로그)

## 3) 권한 모델(초안)
역할(Role)
- `USER`: 일반 사용자
- `DEPT_MANAGER`: 부서 관리자(부서/그룹 운영 일부 권한)
- `ADMIN`: 회사 관리자(전사 정책/사용자/조직 관리)
- `SUPER_ADMIN`: 시스템 운영자(멀티테넌트/복구/강제 조치)

권한 원칙
- 최소 권한 원칙(Least Privilege)
- 테넌트(회사) 경계 강제
- 모든 API는 서버에서 권한 재검증(클라이언트 신뢰 금지)

## 4) 시스템 구성(권장)
모노레포 기준으로 아래 구성을 권장한다.
- `apps/electron-user`: 사용자 앱(Electron + React)
- `apps/admin-web`: 관리자 웹(React)
- `apps/api`: 백엔드 API + WebSocket 게이트웨이
- `packages/shared`: 공통 타입/권한 유틸/상수

주요 기술 선택(권장)
- API: Node.js + NestJS 또는 Fastify
- DB: PostgreSQL
- 실시간: WebSocket(Socket.IO 또는 WS)
- 캐시/이벤트 브로커: Redis
- 인증: JWT + Refresh Token + RBAC
- 알림: Electron Notification API

## 5) 실시간/알림 정책
- 일정 생성/수정/삭제 이벤트를 서버에서 발행
- 클라이언트는 권한 범위의 이벤트만 구독
- 신규 공유 일정 수신 시
  - 일정 목록/캘린더 즉시 갱신
  - PC 알림 발송(중복 방지 키 사용)
- 오프라인 복귀 시 누락 이벤트 동기화(마지막 수신 시점 기준)

## 6) 관리자 기능(필수 제안)
Admin 페이지에는 아래 기능을 포함한다.
- 사용자 관리: 초대, 비활성화, 부서/직책 변경, 일괄 업로드(CSV)
- 조직 관리: 회사/부서 구조 관리, 부서장 지정
- 그룹 관리 모니터링: 공개/비공개 그룹 정책, 비정상 그룹 정리
- 권한 정책 관리: 역할별 권한 템플릿, 예외 권한 승인 플로우
- 일정 감사로그: 생성/수정/삭제/공유 변경 이력 조회
- 공지/시스템 알림: 전사 공지 발행, 긴급 배너
- 보안 관리: 로그인 이력, 세션 강제 만료, 접근 차단(IP/계정)
- 통계 대시보드: DAU, 일정 생성량, 공유 비율, 알림 실패율
- 데이터 관리: 백업/복구 실행 이력, 보존 기간 정책

## 7) 보안 및 개인정보 기준
- 비밀번호/토큰/개인정보는 로그/클라이언트 저장소에 평문 저장 금지
- 민감 필드는 마스킹하여 표시
- API/웹소켓 전 구간 TLS 적용
- 감사로그는 위변조 방지 저장소 또는 보존 정책 적용

## 8) 개발 진행 원칙
- 작은 단위로 단계별 구현 후 체크리스트 기준 검증
- 기능 구현 시 최소 단위 테스트 + 핵심 시나리오 E2E 작성
- 스키마/권한/이벤트 변경 시 마이그레이션 및 롤백 계획 동시 작성
- 큰 변경(의존성, DB 스키마 대규모 개편, 인증 정책 변경, 로깅 정책 변경)은 사전 합의 후 진행

## 9) 완료 기준(Definition of Done)
아래 조건을 충족해야 해당 기능 완료로 본다.
- 요구사항 충족(기능/권한/예외 처리)
- 테스트 통과(단위/통합/E2E)
- 관리자 감사로그 확인 가능
- 문서 업데이트(API/모델/운영 가이드)
- 보안 점검 체크 통과(민감정보/권한 누락 없음)
